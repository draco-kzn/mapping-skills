# Mapping Skills

Claude Code 技能集 —— 赛道 Mapping 方法论，用于系统性梳理产业链、结构化分析投资机会。

## 技能列表

### industry-mapping —— 产业链 Mapping

当用户说"帮我 mapping 一下 XX 赛道"、"做 XX 产业链梳理"、"XX 赛道全景分析"时自动触发。

基于七步法方法论，交付可交互的 Markdown 分析和结构化 Excel。参考量子科技 Mapping 格式。

#### Mapping 七步法

```
用户提出赛道/主题
     │
     ↓
[01] 资料收集 —— 读已有材料 + 网页搜索补充
     │
     ↓
[02] 产业链拆解 —— 按上中下游拆分赛道
     │
     ↓
[03] Startup 深度拆解 —— 对标分类 + 11 维度分析
     │
     ↓
[04] 龙头上市公司梳理 —— 各环节龙头/相关标的
     │
     ↓
[05] 海外对标 —— 国际同赛道 Startup
     │
     ↓
[06] 应用/场景矩阵 —— 场景 × 规模 × 成熟度 × 付费方
     │
     ↓
[07] 投资视角分析 —— 可投性评级/卡脖子/国产替代/优先级
```

#### 三种模式

| 模式 | 涵盖阶段 | 适用场景 |
|------|---------|---------|
| 精简模式 | 01→02→03 | 快速扫描赛道 Startup |
| 标准模式 | 01→02→03→04→06 | 标准 Mapping：产业链 + 上市公司 |
| 完整模式 | 01→02→03→04→05→06→07 | 完整 Mapping：含海外对标 + 投资建议 |

#### 输出物

- **Markdown 报告**: 包含产业链全景图、公司对比、投资建议的结构化文档
- **Excel**: 包含 Startup 总表/海外对标/上游零部件/上市公司/场景矩阵/投资视角，共 6 个 Sheet

#### 每家 Startup 11 项指标

公司名称 | 细分赛道 | 成立/总部 | 创始人/CEO | 技术路线 | 核心产品/参数 | 商业化阶段 | 融资情况 | 团队背景 | 核心优势 | 核心风险

## 文件结构

```
industry-mapping/
├── SKILL.md                              # 入口文件（触发规则、方法论框架、调度逻辑）
├── stages/
│   ├── 01-research.md                    # 桌面研究：资料收集与赛道认知
│   ├── 02-chain.md                       # 产业链拆解：上中下游框架
│   ├── 03-startups.md                    # Startup 深度拆解：11 维对标
│   ├── 04-listed.md                      # 上市公司：A 股标的梳理
│   ├── 05-global.md                      # 海外对标：国际 Startup 扫描
│   ├── 06-market.md                      # 场景矩阵：市场规模量化
│   └── 07-investment.md                  # 投资视角：可投性 + 卡脖子分析
└── templates/
    └── excel-format.md                   # Excel 交付物格式规范与配色
```

## 使用方法

1. 确保该技能已链接到 `~/.claude/skills/` 目录
2. 在对话中输入："帮我 mapping 一下 XX 赛道"
3. Claude Code 将自动加载本技能并按七步法执行

## 自定义

- 每个阶段一个独立 Markdown 文件，可直接编辑修改
- Excel 格式参考 `templates/excel-format.md`，支持 openpyxl 风格化输出
- 通过 SKILL.md 中的调度规则控制各阶段的启用/跳过
